# RS Meets: JS Bug Hunt 🐞

Учебный проект для митапа RS School. Пять мини-приложений на чистом JavaScript, в каждом из которых притаилось ровно **5 багов**. Задача — найти и исправить их.

> ⚠️ Важно: все баги находятся **только в файлах `script.js`**. HTML и CSS трогать не нужно — они подсказывают, как должно работать приложение в финале.

Сложность растёт от первого задания к пятому: от опечаток и неверных операторов до работы с массивами и DOM.

## 🚀 Как начать работу

1. **Сделай форк** этого репозитория к себе на GitHub (кнопка `Fork` в правом верхнем углу страницы репо).
2. Склонируй свой форк к себе на компьютер:
   ```bash
   git clone https://github.com/<твой-логин>/RS-Meets-JS-bugfixing.git
   ```
3. Перейди в папку проекта:
   ```bash
   cd RS-Meets-JS-bugfixing
   ```
4. Создай новую ветку для исправлений:
   ```bash
   git checkout -b fix/bugs
   ```
5. Открой проект в редакторе (например, VS Code):
   ```bash
   code .
   ```
6. Открой `index.html` в браузере (или запусти через Live Server в VS Code) и переходи по ссылкам на задания.
7. Открой DevTools (F12) и начинай охоту на баги — все они в файлах `script.js`.
8. После исправлений закоммить и запушь изменения:
   ```bash
   git add .
   git commit -m "fix: bugs in task 1"
   git push origin fix/bugs
   ```
9. На GitHub открой Pull Request из своей ветки в `main`.

> **Подсказка:** в каждом задании в левом верхнем углу есть жёлтый значок 🐞 — наведи на него курсор, чтобы увидеть список из 5 багов и того, как должно работать в правильном виде.

## 📋 Задания

### Задание 1 ⭐ — Счётчик кликов

Простой счётчик с тремя кнопками: «+», «−», «Сброс». При значении 10 должно появляться поздравление.

| № | Что не так | Как должно быть |
|---|------------|----------------|
| 1 | Кнопка «+» уменьшает значение вместо увеличения | Прибавляет 1 при клике |
| 2 | Кнопка «−» вообще не реагирует | Уменьшает значение на 1 |
| 3 | Кнопка «Сброс» возвращает не к 0 | Возвращает счётчик к 0 |
| 4 | Поздравление не появляется ровно на значении 10 | На значении 10 показывается «🎉 Поздравляем, вы достигли 10!» |
| 5 | Счётчик уходит в отрицательные числа | Не опускается ниже 0 |

### Задание 2 ⭐⭐ — Калькулятор чаевых

Форма с полями: сумма счёта, процент чаевых, количество людей. По кнопке считает сумму чаевых, итог и долю на человека.

| № | Что не так | Как должно быть |
|---|------------|----------------|
| 1 | Кнопка «Посчитать» не реагирует на клик | Запускает функцию расчёта |
| 2 | Числа из полей склеиваются как строки (`1500 + 10 = "150010"`) | Работает арифметика, значения конвертируются в число (`Number()` или `parseFloat()`) |
| 3 | Сумма чаевых получается в 100 раз больше нужной | Процент делится на 100 перед умножением |
| 4 | При пустых полях показывается «NaN» | Появляется сообщение «Заполните все поля корректно» |
| 5 | «На человека» делится на сумму чаевых | Делится на количество людей |

### Задание 3 ⭐⭐⭐ — Список задач (To-Do)

Добавление задач, отметка выполненных кликом, удаление кнопкой. Внизу — счётчик задач.

| № | Что не так | Как должно быть |
|---|------------|----------------|
| 1 | После первой задачи всё ломается, массив превращается в число | `push` мутирует массив, его возвращаемое значение присваивать обратно не нужно |
| 2 | Можно добавить пустую строку или строку из пробелов | Показывается ошибка «Введите текст задачи» (использовать `trim()`) |
| 3 | Кнопка «Удалить» ничего не удаляет | Задача удаляется из массива и со страницы (`splice(i, 1)`) |
| 4 | Клик по задаче не зачёркивает её | Класс `done` переключается, список перерисовывается |
| 5 | Старые элементы остаются — список дублируется при каждом рендере | Перед рендером список очищается (`innerHTML = ''`) |

### Задание 4 ⭐⭐⭐⭐ — Галерея со слайдером

Главная картинка, стрелки «‹» и «›», ряд миниатюр. Активная миниатюра подсвечивается жёлтой рамкой.

| № | Что не так | Как должно быть |
|---|------------|----------------|
| 1 | После загрузки нет картинки и подписи | Сразу отображается первый слайд |
| 2 | Стрелка «вправо» уходит в `undefined` после последнего слайда | Зацикливается на первый (используется `% slides.length`) |
| 3 | Стрелка «влево» ломается на первом слайде | Зацикливается на последний |
| 4 | Все миниатюры показывают один и тот же слайд (последний) | Каждая миниатюра показывает свою картинку — нужно использовать `let` вместо `var` в цикле |
| 5 | Активная миниатюра не подсвечивается рамкой | Класс `active` снимается со всех и ставится на текущую |

### Задание 5 ⭐⭐⭐⭐⭐ — Каталог товаров

Каталог с поиском по названию, фильтром по категории и сортировкой по цене. Вверху счётчик найденных товаров.

| № | Что не так | Как должно быть |
|---|------------|----------------|
| 1 | Список всегда пустой при любых фильтрах | В колбэке `filter` забыт `return` — нужно вернуть результат проверок |
| 2 | Поиск работает только при точном совпадении регистра | Сравнение через `.toLowerCase()` с обеих сторон |
| 3 | После сортировки «Цена ↓» опция «По умолчанию» не возвращает исходный порядок | Сортировать копию массива (`result.slice().sort(...)` или `[...result].sort(...)`), а не исходный `products` |
| 4 | Старые карточки не убираются — список дублируется | Перед рендером очищать `listEl.innerHTML = ''` |
| 5 | Счётчик «Найдено: N» показывает общее число товаров, а не отфильтрованных | Показывать `result.length`, а не `products.length` |
