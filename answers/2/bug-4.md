# Баг 4 — При пустых полях выводится «NaN»

## Что происходит

Если нажать «Посчитать» с незаполненными полями, вместо понятного сообщения об ошибке в блоке результата появляется «NaN» (Not a Number).

## Почему так происходит

`NaN` появляется, когда пытаешься сделать математику с нечисловым значением. Пустая строка `""` при умножении даёт `NaN`:

```js
// Если поле пустое:
const bill = parseFloat(''); // NaN
const tipAmount = NaN * 10;  // NaN
```

В коде нет никакой проверки перед вычислениями.

## Как исправить

Добавь проверку с помощью `isNaN()` или просто на falsy-значение перед расчётом:

```js
// ✅ Исправление
function calculate() {
  const bill = Number(billInput.value);
  const tipPercent = Number(tipInput.value);
  const people = Number(peopleInput.value);

  if (isNaN(bill) || isNaN(tipPercent) || isNaN(people) || people === 0) {
    resultEl.innerHTML = '<p>Заполните все поля корректно</p>';
    return;
  }

  // ... вычисления
}
```

## Как лучше

`isNaN(value)` возвращает `true`, если значение невозможно привести к числу. Всегда валидируй данные от пользователя **до** того, как с ними что-то считать. Это называется «валидация на входе» — один из базовых принципов надёжного кода.
