# Баг 5 — «На человека» считается неверно

## Что происходит

Значение «На человека» — явно не то, что ожидается. Например, при счёте 1500 ₽, 10% чаевых и 3 людях результат абсурдный.

## Почему так происходит

`perPerson` делится на `tipAmount` (сумму чаевых) вместо `people` (количество людей):

```js
// ❌ Баг
const perPerson = total / tipAmount;
// 1650 / 150 = 11 — бессмысленное число
```

## Как исправить

```js
// ✅ Исправление
const perPerson = total / people;
// 1650 / 3 = 550 ₽ — верно
```

## Как лучше

Такие баги легко поймать, если перед написанием кода прописать формулу словами:
- Итого = счёт + чаевые
- На человека = **итого** / **количество людей**

Это называется «проверить логику вслух». Если сложно объяснить формулу словами — значит код, скорее всего, неверен.
