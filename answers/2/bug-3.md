# Баг 3 — Сумма чаевых в 100 раз больше нужной

## Что происходит

Для счёта 1500 ₽ и 10% чаевые считаются как 15000 ₽ вместо 150 ₽.

## Почему так происходит

Пользователь вводит процент как целое число — `10` означает 10%. Чтобы получить десятичную долю, нужно разделить на 100. В коде этого деления нет:

```js
// ❌ Баг
const tipAmount = bill * tipPercent;
// 1500 * 10 = 15000 — в 100 раз больше!
```

## Как исправить

```js
// ✅ Исправление
const tipAmount = bill * (tipPercent / 100);
// 1500 * (10 / 100) = 1500 * 0.1 = 150 ₽
```

## Как лучше

Скобки здесь не обязательны с точки зрения приоритета операций (`*` и `/` равны), но они делают код понятнее — сразу видно, что `tipPercent / 100` это «перевод процента в долю». Явность всегда лучше, чем надежда на то, что читатель знает порядок операций.
