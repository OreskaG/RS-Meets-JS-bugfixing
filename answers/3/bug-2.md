# Баг 2 — Можно добавить пустую задачу

## Что происходит

Если нажать «Добавить» с пустым полем или полем из одних пробелов — в список добавляется пустая задача вместо сообщения об ошибке.

## Почему так происходит

Проверка использует `text == ''` — она ловит только полностью пустую строку. Строка из пробелов `"   "` не равна `""`, поэтому проходит проверку:

```js
// ❌ Баг
const text = input.value;
if (text == '') { // "   " != "" — проверка не срабатывает
  errorEl.textContent = 'Введите текст задачи';
  return;
}
```

## Как исправить

Метод `.trim()` убирает пробелы с обоих концов строки. Если после trim строка пустая — значит пользователь ввёл только пробелы:

```js
// ✅ Исправление
const text = input.value.trim();
if (text === '') {
  errorEl.textContent = 'Введите текст задачи';
  return;
}
```

Заодно стоит использовать `===` вместо `==` — строгое сравнение надёжнее.

## Как лучше

Хорошая привычка — всегда делать `.trim()` перед валидацией текстового ввода. Пользователи часто случайно добавляют пробелы, и приложение должно обрабатывать это корректно. Так же можно сохранять `text` уже с `trim()` — тогда и в список попадёт текст без лишних пробелов.
