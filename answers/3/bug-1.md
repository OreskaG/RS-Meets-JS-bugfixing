# Баг 1 — После первой задачи приложение ломается

## Что происходит

Добавляешь первую задачу — она появляется. Пробуешь добавить вторую — приложение перестаёт работать, задачи больше не добавляются.

## Почему так происходит

Метод `push` **изменяет** массив и **возвращает** его новую длину (число). В коде результат `push` присваивается обратно в `todos`:

```js
// ❌ Баг
todos = todos.push({ text: text, done: false });
// После первой задачи: todos = 1 (число!)
// После второй попытки: 1.push(...) — ошибка, у числа нет метода push
```

После первого добавления `todos` перестаёт быть массивом и становится числом `1`. Следующий вызов `.push` падает с ошибкой.

## Как исправить

`push` не нужно присваивать — он изменяет оригинальный массив напрямую:

```js
// ✅ Исправление
todos.push({ text: text, done: false });
```

## Как лучше

Запомни: **методы, которые изменяют массив на месте** (`push`, `pop`, `splice`, `sort`, `reverse`), не нужно присваивать — они работают с самим массивом. Возвращаемое ими значение нужно только если оно тебе специально нужно (например, `pop` возвращает удалённый элемент).
