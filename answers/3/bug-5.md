# Баг 5 — Список задач дублируется

## Что происходит

При добавлении каждой новой задачи список не обновляется — старые задачи остаются, новые добавляются снизу. После трёх добавлений в списке уже 6 элементов, потом 9 и т.д.

## Почему так происходит

Функция `render()` создаёт новые `<li>` и добавляет их в `listEl` через `appendChild`, но никогда не очищает список перед этим:

```js
// ❌ Баг
function render() {
  // Нет очистки! Старые элементы остаются в DOM
  for (let i = 0; i < todos.length; i++) {
    // ...
    listEl.appendChild(li); // просто добавляет в конец
  }
}
```

## Как исправить

Добавь очистку списка в самом начале `render()`:

```js
// ✅ Исправление
function render() {
  listEl.innerHTML = ''; // очистить перед отрисовкой
  for (let i = 0; i < todos.length; i++) {
    // ...
    listEl.appendChild(li);
  }
}
```

## Как лучше

`innerHTML = ''` — простой и распространённый способ очистить контейнер. Это работает хорошо для небольших списков. Для больших приложений существуют более эффективные подходы (например, виртуальный DOM во фреймворках), но для начала `innerHTML = ''` — правильное решение.
