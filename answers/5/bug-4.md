# Баг 4 — При каждом поиске товары дублируются

## Что происходит

При вводе текста в поиск или выборе категории список не обновляется — старые карточки остаются, новые добавляются снизу.

## Почему так происходит

Функция `update()` добавляет карточки через `appendChild`, но не очищает контейнер перед этим:

```js
// ❌ Баг
function update() {
  // Нет очистки! Старые карточки остаются в DOM
  result.forEach(function (p) {
    const li = document.createElement('li');
    // ...
    listEl.appendChild(li); // добавляет в конец — дублирование!
  });
}
```

## Как исправить

```js
// ✅ Исправление
function update() {
  listEl.innerHTML = ''; // очистить перед рендером
  result.forEach(function (p) {
    const li = document.createElement('li');
    // ...
    listEl.appendChild(li);
  });
}
```

## Как лучше

Это та же самая проблема, что и в задаче 3 (todo-список). Это очень частый паттерн ошибки при ручном управлении DOM. Правило: если ты перерисовываешь весь список — очищай контейнер в начале. Альтернатива — полностью заменять `innerHTML` строкой, собранной из шаблонных литералов, что сразу очищает и заполняет в одно действие.
