# Баг 5 — Счётчик найденных товаров всегда показывает одно число

## Что происходит

Счётчик «Найдено: N» всегда показывает одно и то же число (9 — общее количество товаров), независимо от применённых фильтров.

## Почему так происходит

В счётчик передаётся длина оригинального массива `products`, а не отфильтрованного `result`:

```js
// ❌ Баг
foundEl.textContent = 'Найдено: ' + products.length;
// products.length = 9 всегда, даже если result содержит 3 товара
```

## Как исправить

```js
// ✅ Исправление
foundEl.textContent = 'Найдено: ' + result.length;
// result — отфильтрованный массив, его длина соответствует тому, что отображается
```

## Как лучше

Всегда показывай данные из **того же источника**, который используется для отрисовки. Если список строится из `result` — и счётчик должен считать `result.length`. Это помогает избежать рассинхронизации между данными и UI. Хорошей практикой будет обновлять счётчик и список в одной и той же функции, чтобы они всегда были согласованы.
