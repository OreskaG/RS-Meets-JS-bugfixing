# Баг 3 — После сортировки «По умолчанию» не работает

## Что происходит

После выбора сортировки по цене (дорогие или дешёвые) вернуться к исходному порядку нельзя. Опция «По умолчанию» показывает товары в том же отсортированном порядке.

## Почему так происходит

Метод `.sort()` **изменяет оригинальный массив**. После сортировки `products` теряет исходный порядок навсегда:

```js
// ❌ Баг
if (sort === 'asc') {
  result = products.sort((a, b) => a.price - b.price);
  // products теперь отсортирован по возрастанию!
  // При следующем вызове update() products уже не в исходном порядке
}
```

## Как исправить

Создай копию массива перед сортировкой, чтобы оригинал не менялся:

```js
// ✅ Исправление
if (sort === 'asc') {
  result = result.slice().sort((a, b) => a.price - b.price);
  // result.slice() создаёт новый массив — products остаётся нетронутым
}
```

`.slice()` без аргументов возвращает поверхностную копию массива.

## Как лучше

Запомни, какие методы массивов **мутируют** оригинал, а какие — нет:
- **Мутируют**: `sort`, `reverse`, `splice`, `push`, `pop`, `shift`, `unshift`
- **Не мутируют**: `filter`, `map`, `slice`, `concat`, `find`

Если тебе нужен отсортированный массив без изменения оригинала — всегда делай `.slice().sort(...)` или используй современный метод `.toSorted()` (ES2023).
