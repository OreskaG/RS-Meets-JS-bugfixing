# Баг 4 — Все миниатюры открывают одну и ту же картинку

## Что происходит

При клике на любую миниатюру открывается одно и то же (последнее) изображение, независимо от того, какую миниатюру нажал.

## Почему так происходит

Это классическая проблема **замыкания переменной `var` в цикле**.

```js
// ❌ Баг
for (var i = 0; i < slides.length; i++) {
  const t = document.createElement('img');
  t.addEventListener('click', function () {
    show(i); // какой i? тот, что снаружи цикла!
  });
}
// После цикла: i === 6
// Все обработчики замкнулись на одну и ту же переменную i
// При клике любой миниатюры вызывается show(6) — slides[6] = undefined
```

`var` не создаёт новую переменную на каждой итерации — все обработчики ссылаются на **одну** переменную `i`, которая после цикла равна `6`.

## Как исправить

Замени `var` на `let`:

```js
// ✅ Исправление
for (let i = 0; i < slides.length; i++) {
  const t = document.createElement('img');
  t.addEventListener('click', function () {
    show(i); // теперь каждая итерация имеет свой i
  });
}
```

`let` создаёт **новую** переменную `i` для каждой итерации цикла, поэтому каждый обработчик замыкается на своё собственное значение.

## Как лучше

Это одна из главных причин, почему `var` считается устаревшим и не рекомендуется к использованию. В современном JavaScript всегда используй `let` или `const` — они ведут себя предсказуемо. `var` остался в языке ради обратной совместимости, но новый код на нём не пишут.
